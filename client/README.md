# My React App

I built this project using Create React App.

## Available Scripts

In the project directory, you can run these commands:

### `npm start`

This runs the app in development mode.\
You can open [http://localhost:3000](http://localhost:3000) to see it in your browser.

The page will automatically reload if you make any changes. You'll also see any lint errors in the console.

### `npm test`

This launches the test runner in interactive watch mode.\
For more information, you can see the section on [running tests](https://facebook.github.io/create-react-app/docs/running-tests).

### `npm run build`

This command builds the app for production and puts the output in the `build` folder.\
It bundles React in production mode and optimizes everything for the best performance.

The build is minified, and the filenames include hashes, so it's ready to be deployed!

You can find more information in the section on [deployment](https://facebook.github.io/create-react-app/docs/deployment).

### `npm run eject`

**Heads up: this is a one-way operation. Once you `eject`, you can't go back!**

If you're not happy with the build tools and configuration, you can `eject` at any time. This command removes the single build dependency from the project.

Instead, it copies all the configuration files and dependencies (like webpack, Babel, ESLint, etc.) directly into the project, so you have full control over them. All the commands except `eject` will still work, but they'll point to the copied scripts so you can customize them. At that point, you're on your own.

You don't ever have to use `eject`. The default setup is great for most projects, and you shouldn't feel like you have to use this feature. But, I understand that this tool wouldn't be as useful if you couldn't customize it when you're ready.
